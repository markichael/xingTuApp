# 实现图片编辑后跳转到分享结果页面

## 目标
1. 图片编辑后跳转到分享结果页面
2. 右上角按钮返回主页
3. 左上角返回编辑页（同时传递照片URL）

## 实现步骤

### 1. 修改 HomeScreen.kt
- 添加 `shareResultUri` 状态变量，用于管理分享结果页面的显示
- 添加 `shareResultSource` 状态变量，用于记录来源页面（普通编辑、AI修图、拼图）
- 修改路由逻辑，添加分享结果页面的显示条件

### 2. 修改各个编辑页面
- **EditorScreen.kt**：修改保存逻辑，保存成功后设置 `shareResultUri` 和 `shareResultSource`，而不是直接返回
- **FaceBeautyScreen.kt**：同样修改保存逻辑
- **CollageScreen.kt**：同样修改保存逻辑

### 3. 完善 ShareResultScreen.kt
- 添加 `imageUri` 参数，用于接收保存的图片URI
- 添加 `source` 参数，用于记录来源页面
- 实现返回编辑页的逻辑，根据来源页面跳转到对应的编辑页面

### 4. 确保导航逻辑正确
- 分享结果页面右上角按钮直接返回主页（清空所有状态）
- 分享结果页面左上角按钮返回对应的编辑页面（传递图片URI）

## 技术要点
- 使用 Compose 状态管理实现页面间导航
- 确保图片URI在页面间正确传递
- 保持状态的一致性和可恢复性

## 文件修改清单
1. `HomeScreen.kt` - 添加状态管理和路由逻辑
2. `EditorScreen.kt` - 修改保存逻辑
3. `FaceBeautyScreen.kt` - 修改保存逻辑
4. `CollageScreen.kt` - 修改保存逻辑
5. `ShareResultScreen.kt` - 完善页面逻辑

## 预期效果
- 编辑图片后点击保存，跳转到分享结果页面
- 分享结果页面显示保存的图片
- 点击右上角按钮返回主页
- 点击左上角按钮返回编辑页面，继续编辑当前图片
